# HealthLake CI — Dashboard Wireframes

## Objective

This document describes the dashboard strategy for each decision-making level.

## Dashboard Personas

[Insert Draw.io Dashboard Wireframes]

## 1. Ministry Dashboard

Purpose: National healthcare supervision.

KPIs:

- Total hospitals
- National bed occupancy
- Critical patients
- Admissions by region
- Equipment availability

## 2. Regional Dashboard

Purpose: Regional healthcare performance monitoring.

KPIs:

- Regional admissions
- Hospital comparison
- Emergency activity
- Bed capacity by hospital
- Critical alerts by facility

## 3. Hospital Dashboard

Purpose: Hospital operations management.

KPIs:

- Current bed occupancy
- Admissions today
- Discharges today
- Emergency waiting time
- Equipment utilization

## 4. Clinical Monitoring Dashboard

Purpose: Real-time patient monitoring.

KPIs:

- Heart rate
- Oxygen saturation
- Temperature
- Blood pressure
- Critical alerts

## Design Principle

Each dashboard is aligned with a specific persona, security level, and decision-making responsibility.
